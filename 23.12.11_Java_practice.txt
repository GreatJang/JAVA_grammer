시간복잡도
표현방법 :O(n) => 빅오 표기법.
선택정렬은 n^2(제곱)만큼의 복잡도를 가지고 있다. = 느리다.

코테 문제는 맞았는데, 시간초과되서 효율성이 떨어져서 시간초과되는 경우가 있음.
단순한 for문의 경우 n^2만큼의 복잡도를 가지고 있기 때문에 느려서 시간초과 되는 경우가 있다.
그럴경우 다른 알고리즘을 사용해야한다.

n^2, n, log2n8(2^3) = 3
시간복잡도 문제가 생겼다면 log2n의 복잡도를 가지고 있는 알고리즘을 사용해야한다.
log2n의 복잡도를 가지고이 있는 알고리즘 들이 있음(기억해야함).
n이 백만이면 log2n은 대략20정도. 엄청난 차이.

이진검색을 하면 log2n의 복잡도를 가질 수 있다.

버블정렬

-검색
    - for문+break를 통한 직접구현
    - 이진검색
        - 이진 검색 알고리즘이란 가운데에 있는 항목을 키값과 비교하여 다음 탐색 위치를 결정
        - binarySearch()
        - sort() 메소드 등을 사용하여 미리 정렬되어 있어여만 이진검색 제대로 동작

    - 배열복사
        - copyOf(대상배열, endIndex)
            - 처음부터 endIndex까지
            - 대상배열을 넘어서는 경우 초기값으로 세팅되어 endindex만큼의 배열의 길이로 세팅

-2차원 배열
    - 배열안에 또다른 배열이 있는 경우
    - 선언 방법 예시
        - int[][] arr = new int[2][3];
        - int[][] arr2 = new int[2][]; // 가변배열 뒤에를 비워두고 나중에 할당 가능.
        - int[][] arr3 = {{1,2,3},{1,2,3}}; // 리터럴 방식
        - int[][] arr4 = {{10,20},{10,20,30},{10,20,30,40}}; // 리터럴 방식

- 컬렉션 프레임워크
    - 다수의 데이터를 처리하는 클래스의 집합
    - List 인터페이스, Set 인터페이스, Map 인터페이스 등

- 자료형 - 리스트(list)
    - java에서 LIST는 여러 개의 요소를 저장하는 자료구조로서 배열과 다르게 요소의 추가/삭제가 가능

    - 기본문법
        - ArrayList<String> pitches = new ArrayList<String>(); // ArrayList = 구현체
        - 인터페이스 = 구현체
            - ArrayList<String> pitches = new ArrayList<>();
            - 일반적으로 뒷부분의 자료형은 생략
        - List<String> pitches = new ArrayList<>();
        - 가장 일반적인 선언방식으로 앞부분의 Arraylist대신에~
